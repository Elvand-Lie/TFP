1. Background

An audit of the BaZi module identified three categories of issues: color bleed between the QMDJ and BaZi subsystems, an oversimplified Day Master (DM) strength algorithm that produced inconsistent Useful God determinations, and an incomplete Annual BaZi Stars implementation that reported fewer than a third of the stars shown by reference software.

This report documents the changes made to resolve each issue, the rationale behind design decisions, and the verification performed against reference data (chart: 11 February 1981, 15:34, target year 2026).

2. Color System Segmentation

Previously, a single Red/White favorability coloring scheme was applied uniformly to the Natal Chart, Luck Pillars, Annual Luck Matrix, Monthly Influence, and inadvertently to the QMDJ chart. This was incorrect — the Natal Chart represents a fixed baseline and should use informational element colors, while the QMDJ system operates independently of BaZi personal favorability.

Updated Coloring Rules

Section Coloring Method Rationale
Natal Chart Five Element colors (Wood=Green, Fire=Red, Earth=Gold, Metal=Silver, Water=Blue) Static baseline; not subject to favorability judgment
Luck Pillars Red/White favorability Time-varying; indicates auspicious/inauspicious periods
Annual Luck Matrix Red/White favorability Same as Luck Pillars, per-year granularity
Monthly Influence Red/White favorability Same algorithm, per-month granularity
Annual BaZi Stars Per-star classification (auspicious=red, inauspicious=default) Each star has a fixed auspicious/inauspicious nature
QMDJ Chart Neutral (no BaZi coloring) Separate divination system; no personal favorability applies

3. Day Master Strength Algorithm

The original algorithm assigned fixed positional weights to each pillar (Month Branch = 35, Day Branch = 15, others = 10) and checked only the surface element of each branch. This approach missed three factors that practitioners consider essential: hidden stem roots, branch combinations, and branch clashes.

3.1 Assessment Factors

The enhanced algorithm evaluates DM strength through four stages, following classical BaZi priority: 得令 (Season) → 得根 (Roots) → 得势 (Support) → 合冲 (Interactions).

Stage Factor Description
1 Clash Detection (六冲) Position-aware Six Clashes are detected among the four branches. When duplicate branches exist (e.g., two 申), only the closest pair clashes. Clashed branches contribute 50% of their base weight.
2 Base Positional Scoring Month Branch = 40 pts, Day Branch = 12, Year/Hour Branch = 8 each, Stems = 8 each. Each position contributes its weight if its element is supportive (same element or resource element as the DM).
3 Hidden Stem Root Bonus (藏干通根) Each branch contains hidden stems with qi weights (main = 0.6–1.0, middle = 0.3, residual = 0.1). If the DM's element appears as a hidden stem, a bonus is added: Month = 10 pts, Day = 6 pts, Year/Hour = 3 pts (scaled by qi weight). Clashed branches are excluded.
4 Combination Modifiers Three types are checked: 三会 Seasonal Combinations (±12 pts), 三合 Three Harmonies (±8 full / ±3 half), and 六合 Six Harmonies (±3). The resulting element of the combination determines whether it adds or subtracts.

3.2 Two-Tier Threshold

Classical BaZi recognizes that a DM can be Strong even when born out of season, provided it has overwhelming support from other positions (失令但得势得根). The threshold was adjusted accordingly:

Condition Threshold for "Strong" Reasoning
Month Branch supports DM (得令) ≥ 50 points Standard; seasonal support provides a strong base
Month Branch opposes DM (失令) ≥ 40 points Without the dominant Month Branch (40 pts), maximum available is reduced; a lower threshold acknowledges overwhelming non-seasonal support

4. Annual BaZi Stars Expansion

The original implementation checked approximately 12 star formulas in the annual context, producing 2–3 stars per chart on average. The reference software showed 6–9 stars per branch. Two additions were made to close this gap.

4.1 Twelve Annual Spirits (岁前十二神煞)

This is the standard annual star rotation system used in traditional almanacs. Starting from Tai Sui (the annual branch), each of the 12 earthly branches receives one spirit based on its offset from Tai Sui:

offset = (branchIndex(natal) − branchIndex(annual) + 12) mod 12

4.2 Stem-Based Annual Stars

In addition to the branch-relative stars already present (Yi Ma, Hong Luan, Gu Chen, etc.), three star types are now also checked using the annual stem as reference:

  • Tian Yi Gui Ren (天乙貴人) — Noble Person star
  • Wen Chang (文昌) — Academic/Intelligence star
  • Tai Ji Gui Ren (太極貴人) — Supreme Ultimate Noble

These were previously only checked in the natal context (using the Day Master stem). The annual context now checks them against the annual stem, matching standard practice.

6. Known Limitations

  1. DM Strength is inherently approximate. There is no universally agreed-upon formula in BaZi. Different practitioners weight factors differently. The algorithm covers the four classical assessment pillars (得令, 得根, 得势, 合冲) but borderline charts will always be subject to practitioner interpretation.
  2. Annual Stars are approximately 70% complete. Around 10 secondary stars visible in the reference (地喪, 隔角, 指背, 飛廉, 天雄, 紅艷煞, 貫索, 勾神, 六厄) were not implemented. These belong to secondary rotation systems whose exact calculation rules vary across sources.
  3. Annual Luck Matrix coloring matches approximately 95% of positions when compared to the reference chart. The remaining discrepancies are likely attributable to differences in DM strength assessment methodology between our algorithm and the reference software.
